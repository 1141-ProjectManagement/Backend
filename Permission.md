# 系統權限與業務邏輯規劃 (Permission Specification)

本文檔根據資料庫設計，詳細闡述了系統中不同角色（Role）所擁有的權限（Permission）及其對應的業務邏輯。

---

## 權限設計總覽

本系統的權限控制基於「角色-權限」模型（Role-Based Access Control, RBAC）。每個使用者被指派一個角色，而每個角色則被授予一組特定的操作權限。這確保了使用者只能訪問其職責範圍內的功能。

權限名稱採用 `資源.動作`（例如 `course.create`）的格式，使其語意清晰且易於管理。

---

## 各角色權限與業務邏輯詳解

以下將逐一分析每個角色的權限和其在系統中的具體職責。

### 1. Student (學生)

**角色描述**：系統中的核心使用者，主要進行課程學習與查詢相關活動。

| 權限代碼 (Permission Name) | 權限描述 | 業務邏輯與應用場景 |
| :--- | :--- | :--- |
| `profile.view.own` | 查看自己的個人資料 | 學生可以登入系統後，在「個人中心」頁面看到自己的學號、姓名、生日等資訊。 |
| `profile.edit.own` | 修改自己的個人資料 | 學生可以修改自己的聯絡方式、密碼等非核心個資。學號等核心資訊通常不允許修改。 |
| `course.view.catalog` | 查看可用的課程目錄 | 學生可以瀏覽全校開設的所有課程列表，查看課程描述、學分和授課教師。 |
| `course.enroll.self` | 自己註冊選修一門課程 | 在選課期間，學生可以對開放的課程點擊「選課」按鈕，將自己加入到課程的選課名單中。 |
| `grade.view.own` | 查看自己的成績 | 學生可以查看自己所有已修課程的成績。 |

### 2. Teacher (教師)

**角色描述**：負責課程教學與學生輔導。

| 權限代碼 (Permission Name) | 權限描述 | 業務邏輯與應用場景 |
| :--- | :--- | :--- |
| `profile.view.own` | 查看自己的個人資料 | 教師可以查看自己的教職員編號、姓名、年齡等個人資訊。 |
| `profile.edit.own` | 修改自己的個人資料 | 教師可以修改自己的聯絡方式、密碼、辦公室地點等。 |
| `course.view.catalog` | 查看可用的課程目錄 | 教師可以瀏覽全校的課程，以了解其他課程的開設情況。 |
| `grade.update` | 為學生輸入或修改成績 | 對於自己所教授的課程，教師可以為名單中的每一位學生輸入或修改期末成績。 |
| `grade.view.all` | 查看課程中所有學生的成績 | 教師可以查看並導出自己所教授課程的全體學生修課名單與成績。 |

### 3. Student Affairs Admin (學務管理員)

**角色描述**：負責管理全校學生的學籍與選課相關事宜。

| 權限代碼 (Permission Name) | 權限描述 | 業務邏輯與應用場景 |
| :--- | :--- | :--- |
| `profile.view.own` | 查看自己的個人資料 | 管理員的基本權限。 |
| `profile.edit.own` | 修改自己的個人資料 | 管理員的基本權限。 |
| `student.create` | 建立學生帳號 | 在管理後台為新生或轉學生建立新的學生帳號及個人檔案。 |
| `student.view` | 查看學生資料 | 可以查詢、檢視任何一位學生的詳細個人資料與學籍狀態。 |
| `student.update` | 修改學生資料 | 當學生資料有誤或需要變更時（如更改姓名、學籍狀態），管理員有權限進行修改。 |
| `student.delete` | 刪除學生帳號 | 處理學生退學、畢業等情況，將學生帳號歸檔或刪除。 |
| `enrollment.manage` | 手動處理學生選課/退選 | 處理特殊情況下的選課需求，例如為學生手動加選或退選課程。 |
| `grade.view.all` | 查看課程中所有學生的成績 | 當需要處理成績單或學籍警示時，可以查看任一課程的學生名單與成績。 |

### 4. Course Affairs Admin (課務管理員)

**角色描述**：負責管理全校的課程開設、排程與教師分配。

| 權限代碼 (Permission Name) | 權限描述 | 業務邏輯與應用場景 |
| :--- | :--- | :--- |
| `profile.view.own` | 查看自己的個人資料 | 管理員的基本權限。 |
| `profile.edit.own` | 修改自己的個人資料 | 管理員的基本權限。 |
| `teacher.view` | 查看教師資料 | 為了安排課程，需要能查看所有教師的資訊。 |
| `course.create` | 建立新課程 | 在管理後台根據教學計畫建立新的課程。 |
| `course.update` | 修改課程資訊 | 修改課程的名稱、描述、學分等資訊。 |
| `course.delete` | 刪除課程 | 將不再開設的課程從系統中移除。 |
| `course.assign` | 為課程分配教師 | 將建立好的課程指派給特定的教師來授課。 |

### 5. Teacher HR Admin (人資管理員)

**角色描述**：負責管理教師的人事資料。

| 權限代碼 (Permission Name) | 權限描述 | 業務邏輯與應用場景 |
| :--- | :--- | :--- |
| `profile.view.own` | 查看自己的個人資料 | 管理員的基本權限。 |
| `profile.edit.own` | 修改自己的個人資料 | 管理員的基本權限。 |
| `teacher.create` | 建立教師帳號 | 為新進的教師建立帳號和人事檔案。 |
| `teacher.view` | 查看教師資料 | 查詢、檢視任何一位教師的詳細人事資料。 |
| `teacher.update` | 修改教師資料 | 更新教師的職級、薪資、聯絡方式等資訊。 |
| `teacher.delete` | 刪除教師帳號 | 處理教師離職、退休等情況。 |

### 6. Super Administrator (超級管理員)

**角色描述**：擁有系統的最高權限，負責系統維護與管理員帳號管理。

| 權限代碼 (Permission Name) | 權限描述 | 業務邏輯與應用場景 |
| :--- | :--- | :--- |
| (全部權限) | (全部權限) | 超級管理員擁有以上所有角色的權限，並且額外擁有以下權限： |
| `admin.create` | 建立管理員帳號 | 可以建立新的學務、課務、人資管理員帳號。 |
| `admin.update` | 修改管理員帳號與權限 | 可以修改其他管理員的資料，並調整其角色（進而調整其權限）。 |
| `admin.delete` | 刪除管理員帳號 | 刪除不再需要的管理員帳號。 |
| `system.settings.edit` | 修改系統級設定 | 調整系統的全域設定，例如選課開放時間、學期設定等。 |
